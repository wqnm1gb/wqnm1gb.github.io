为什么需要new ArrayList

这个解释很好

注意啊：别再犯错了！

    <insert id="save">
        insert into setmeal_dish (setmeal_id,dish_id,name,price,copies)
        values
        <foreach collection="setmealDishes" open="(" close=")" separator="," item="setmealDishe">
            #{setmealDishe.setmealId},#{setmealDishe.dishId},#{setmealDishe.name},#{setmealDishe.price},#{setmealDishe.copies}
        </foreach>
    </insert>

setmealDishe 用foreach遍历的时候，如果需要访问里边的元素，需要使用setmealDishe.setmealId的形式来访问

在分页查询套餐的时候，需要多表查询，因为最后返回的还有分类名字！！！

    <select id="pageQuery" resultType="com.sky.vo.SetmealVO">
        select s.*,c.name category_name from setmeal as s left join category as c
            on s.category_id = c.id
        <where>
            <if test="name != null ">s.name like concat('%',#{name},'%') </if>
            <if test="categoryId != null">s.category_id = #{categoryId}</if>
            <if test="status != null">s.status = #{status}</if>
        </where>
        order by s.create_time desc
    </select>

test = “name != null” 这里的name不需要s.name 因为这是判断类里边的，不是数据库里边的

请注意 update set后边的标签判断的后边需要加逗号

一个错误，在insert语句中

这样写是正确的，foreach标签里边，需要写一个括号()

    <insert id="save">
        insert into setmeal_dish (setmeal_id,dish_id,name,price,copies)
        values
        <foreach collection="setmealDishes" separator="," item="setmealDishe">

这样写是错误的

    <insert id="save">

        <foreach collection="setmealDishes" separator="," item="setmealDishe" open= "(" close=")">

        </foreach>

生成的语句是（括号在整体的地方）

INSERT INTO setmeal_dish (...)

  (1, 101, 'A', 10.0, 2, 1, 102, 'B', 15.0, 3);
