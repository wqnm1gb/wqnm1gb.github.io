Controller层接收参数的时候需不需要加@RequestBody？

如果是接受的Json格式的数据，那么需要加，如果是接受GET的参数，即使是接受到类里，也可以不用加

注意看分页查询的时候返回的结果啊，还有个Total参数，所以一定需要定义一个单独的对象接受PageResult

Mybatis 在xml中>是合法的 <是非法的，需要改成<

前端请求的信息，服务器响应回来的可以不一定和接口文档的响应信息一模一样，可以有些多的字段，且字段值为null

如何在插入数据的时候有可选字段，在values后采用if标签

    <insert id="add">
        insert into clazz(name, room, begin_date, end_date, master_id, subject,create_time,update_time) 
        values ( #{name},
                #{room},
                #{beginDate},
                #{endDate},
                <if test="masterId!=null">
                #{masterId},
                </if>
                #{subject},
                #{createTime},
                #{updateTime}
               );
    </insert>

思考一下，这样子是错误的，因为要求有这么多个参数，你不能少了，而且masterId在数据库中定义的本来就是可以为null所以不需要这个if标签即可

如果前端传来的是Json格式的数据，请务必记得用@RequestBody注解标注后接收！！！

自定义异常可以这么定义

public class ClazzNotNullException extends RuntimeException{
    public ClazzNotNullException() {
        super();
    }
    public ClazzNotNullException(String message) {
        super(message);
    }
}

也不一定需要构造器，如果没有构造器的话，会默认调用父类的无参构造器，这时候就不能传递错误信息

专门定义的全局异常处理器

    @ExceptionHandler
    public Result HandlerClazzNotNullException(ClazzNotNullException e){
        log.error("出错了",e);
        return Result.error("对不起, 该班级下有学生, 不能直接删除");
    }

突然发现的一个问题

select count(*) count,dept_id  from dept d left join emp e on d.id = e.dept_id where d.id=11 group by dept_id;

这个语句会把我在emp中表里边部门为null的人也统计上

原因是什么呢，是左表连接

分析一下左表连接

select 字段列表 from 表1 left [outer] join 表2 on 连接条件 …;

这里的左表是表1，也就是说，表1里边的所有数据都会保留，即使右表没有对应的数据，也会显示为null，所以说上边写的把部门为null的人统计上是错的，因为只是右表没有人，所以显示为null，这里最好用右表连接或者内连接，或者换个逻辑，因为已经知道id的值了，根本不需要连接查询，直接在emp表里边找就好了
