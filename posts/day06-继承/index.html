Day06 继承

二月 8, 2025 · 2 分钟 · 970 字 · 冷漠三狗子丶

不仅会退出循环，还会结束整个方法的执行

比如说匹配id的时候，找到id就直接退出方法，没找到id的话在循环语句后边还会有一个没找到id的提示

子类能继承父类的非私有成员（成员变量，成员方法）如果定义的变量是私有的，那么能继承公有的get，set方法

子类对象是由子类和父类多张设计图共同创建出来的对象，所以子类对象是完整的

private     只能本类
缺省         本类，同一个包中的类
protected   本类，同一个包中的类，子孙类中
public      任意位置

private<缺省<protected<public

**java为什么不支持多继承 **

java中的祖宗类object

java中的所有类，要么直接继承object，要么默认继承object，要么间接继承object

如果子类中，出现了重名的成员，会优先使用子类的，如果此时一定要使用父类的，需要加上super关键字

Java的多个类中有且仅有一个可以用public修饰

重写的时候会加上 @Override 在上边，这是方法重写的校验注解（标志），要求方法名称和形参列表必须与被重写的方法一致，否则报错

优点：更安全，更易读

注意事项

1、子类方法重写时候，访问权限一定要大于等于父类方法的权限（public>protected>缺省）

2、重写方法的返回值类型，必须与父类的返回值类型一样或者范围更小

3、私有方法，静态方法不能被重写，如果重写会报错（原因是子类只能继承非私有的成员，所以不能重写）

直接输出对象，会调用对象的toString()方法，返回对象的地址信息

具体是调用父类的有参还是无参由自己决定，如果不管，默认调用无参，如果没有无参，需要自己写一个有参

如何实现调用父类的构造器的？

默认情况下，子类构造器的第一行都是super();（写不写都有），会调用父类的无参构造器

如果父类没有无参构造器，则子类构造器的第一行必须手写一个super()，去指定调用父类的有参构造器

this调用本类中的兄弟构造器

this的作用是，在构造器中调用本类的其他构造器

注意，super()和this()必须写在构造器的第一行，而且两者不能同时出现

public class Person {
    private String name;
    private int age;

    // 第一个构造器
    public Person(String name) {
        this(name, 20); // 调用第二个构造器
    }

    // 第二个构造器
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 其他方法
    public void display() {
        System.out.println("Name: " + name + ", Age: " + age);
    }

    public static void main(String[] args) {
        Person p1 = new Person("Alice");
        p1.display(); // 输出: Name: Alice, Age: 20

        Person p2 = new Person("Bob", 30);
        p2.display(); // 输出: Name: Bob, Age: 30
    }
}
