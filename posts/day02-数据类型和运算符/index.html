2、强制转换，取值范围大的变成取值范围小的

要强制转换的是最终的结果

如果我想把c定义为byte型

byte c = a + b //报错
byte c = (byte)a + b //报错，相当于强制转换a
byte c = (byte)a + (byte)b //报错，相当于强制转换a和b，并不是转换最后的结果
byte c = (byte)(a + b) //正确

在+的过程中，只要有字符串参与，做的都会是拼接的操作

连续+从左到右依次+

+=，-=，*=，/=，%=底层都隐藏了一个强制类型转换

short s = 1;
s + = 1;
//等同于
s = (short)(s+1);

比如第一个不满足了就直接不满足，右边的语句就不会运行了

|| 短路非 结果与|相同，但是有短路效果

比如第一个满足了就全满足，右边的语句就不会运行了

注意事项

&|，无论左边是TRUE还是FALSE，右边都要执行

&& || ，如果左边能确定表达式的结果，右边不执行

关系表达式？表达式1:表达式2

如果关系表达式是真，那么就使用表达式1，如果为假，那就使用表达式2

使用的时候的方法是，max = a>b?a:b //把a，b中的最大值赋给max
