Day05 面向对象编程

一月 2, 2025 · 4 分钟 · 1782 字 · 冷漠三狗子丶

对象#

对象是一种特殊的数据结构，可以用来记住一个事物的数据，从而代表该事物

1、先设计一个对象的模板，也就是对象的设计图：类

2、通过new关键字，每new一次就得到一个新的对象

内存分成了栈内存，堆内存和方法区

方法区存放类文件

new出来的东西叫做对象，会放在堆里边

方法需要使用的时候，会被掉到栈内存中

设计对象的关键就是设计类，对象的模板

构造器#

创建对象的时候，对象会自动的调用构造器

常见应用场景

构造器的作用： 创建对象时，同时完成对对象成员变量（属性）的初始化赋值、

//构造器的语法
class Student{

    public Student(){ //名字必须要和类名相同，可以带参数也可以不带参数，没有返回值
    }
}

构造器的注意事项#

1、类默认就带了一个无参构造器

2、如果类定义了有参构造器，类默认的无参构造器就没有了，此时如果还想用无参构造器，就要自己手写一个无参构造器

this关键字#

this就是一个变量，可以用在方法中，来拿到当前对象（拿的对象地址）

哪个对象调用这个方法，this就拿到哪个对象

**this对象主要用来解决：****变量名称冲突**的问题

成员变量与方法内部变量的名称一样的时候，导致的命名冲突的问题

封装#

面向对象的三大特征：封装，继承，多态

类，方法就是一种封装

封装的设计要求

合理隐藏，合理暴露

如何隐藏：使用private关键字进行修饰成员变量，就只能在本类中被直接访问，其他任何地方不能直接访问

如何暴露： 使用public修饰的get和set方法合理暴露成员变量的取值和

Javabean（实体类）#

是一种特殊类，满足下列要求

1、类中的成员变量全是私有的，并且提供public修饰的getter和setter方法

2、需要提供一个无参数的构造器，有参数的可选

实体类的基本作用：创建他的新对象，存取数据（封装数据）

实体类的应用场景：

实体类的对象只负责数据的存取，而不负责数据的处理，数据的处理交给其他类的对象来完成，已实现数据和业务处理相分离

static关键字#

static修饰变量#

可以用来修饰成员变量和成员方法

类在运行的时候只会被加载一次

成员变量按照有无static修饰可以分成两种：

1、静态变量（类变量）：有static修饰属于类，在计算机中只有一份，会被全部对象共享，以最后的改动为准

2、实例变量（对象的变量）（实例就是对象，对象就是实例（实例化之后就会占用内存） ，类是比他们高一个等级的抽象概念（不占用内存））：无static修饰，属于每个对象的

推荐访问静态变量的方式

类名.静态变量（推荐）

实例名.静态变量（不推荐）

静态变量的应用场景

**如果某个数据只需要一份，且希望能够被共享（访问、修改），则该数据可以定义成静态变量来记住 **

同一个类中，访问静态成员，可以省略类名不写

static修饰方法#

静态方法：有static修饰的成员方法，属于类持有

实例方法：无static修饰的成员方法，属于对象

如果这个方法只是为了做一个功能并且不需要直接访问对象的数据，这个方法直接定义为静态方法

如果这个方法是对象的行为，需要访问对象的数据，这个方法必须定义为实例方法

常见应用场景：做工具类，工具类中的所有方法都是一些静态方法，每一个方法来完成一个功能

使用工具类的好处：提高代码复用，调用方便，提高了开发效率，不会占用内存

工具类没有必要创建对象，可以把构造器进行私有化

public class Tools{
    private Tools(){ //别人无法创建这个类
    }
}

public修饰和没有关键字修饰（只能用这两种修饰符来修饰类）#

public修饰符#

类级别: 如果一个类被声明为 public，它可以被任何地方访问，无论是在同一个包内还是其他包中，只要导入了该类所在的包。

方法/字段级别: 如果类的成员（方法或字段）被声明为 public，那么任何类都可以访问这些成员，无论它们位于哪个包。

没有修饰符（Package-private）#

**类级别: **如果一个类没有使用任何修饰符（即默认情况下），它只能在同一个包内访问。其他包中的类无法访问这个类，即使它们导入了该包。
方法/字段级别: 如果类中的方法或字段没有使用访问修饰符，它们只能在同一个包内访问，而不能被其他包中的类访问。

静态方法，实例方法的注意事项#

1、静态方法中可以**直接访问静态成员****， 不可以直接访问实例成员**

2、实例方法中既可以直接访问静态成员，也可以直接访问实例成员

3、实例方法中可以出现this关键字，静态方法中**不可以出现this关键字**
